The node n<sub>1</sub> has children nodes if there exists an edge from n<sub>1</sub> to each child node. In Figure 1, Node "a" has children nodes "b" and "c".</p> <p>There are two types of edges: directed and undirected. If the edge is directed, then the edge has a specific direction going from start to destination node. Graphs with directed edges are called directed graphs (or <strong>digraph</strong>).</p> <p><a href="/c4x/MITx/6.00.2_2x/asset/files_ps10_files_ps10_directedgraph.gif" target="_blank"><img class="aligncenter size-full wp-image-1163" title="ps10_directedgraph" alt="" src="/c4x/MITx/6.00.2_2x/asset/files_ps10_files_ps10_directedgraph.gif" height="116" width="250" /></a></p> <p><em>Figure 1. Example of a <strong>directed</strong> graph where each edge has a specific direction.</em></p> <p>If the edge is undirected, also known as bidirectional, then it no longer matters which node is the start or destination node because you can traverse the edge from one node to the other in either direction. Essentially, a link in the graph can be represented by a directed edge going from Node "d" to Node "e" and a directed edge going in the reverse direction.</p> <p><a href="/c4x/MITx/6.00.2_2x/asset/files_ps10_files_ps10_undirectedgraph.gif" target="_blank"><img class="aligncenter size-full wp-image-1164" title="ps10_undirectedgraph" alt="" src="/c4x/MITx/6.00.2_2x/asset/files_ps10_files_ps10_undirectedgraph.gif" height="106" width="236" /></a></p> <p><em>Figure 2. Example of an <strong>undirected</strong> graph where each edge is bidirectional.</em></p> <p>An edge can also have a weight. If every edge is associated with a real number (edge weight), then we have weighted graph.</p> <p><a href="/c4x/MITx/6.00.2_2x/asset/files_ps10_files_ps10_weightedgraph.gif" target="_blank"><img class="aligncenter size-full wp-image-1165" title="ps10_weightedgraph" alt="" src="/c4x/MITx/6.00.2_2x/asset/files_ps10_files_ps10_weightedgraph.gif" height="124" width="258" /></a></p> <p><em>Figure 3. Example of an <strong>weighted</strong> graph where each edge has a weight associated with it.</em></p> <p>In a graph theory problem, the <strong>objective function</strong> is the function to be minimized (or maximized). For example, choosing the shortest path for airplane flights is an optimization problem where the objective function is to minimize the distance traveled. The nodes are the destination airports and edges are the presence of airplane routes between airports. We can add additional <strong>constraints</strong> on the problem that must be satisfied such as requiring that the plane only make at most 2 stops along the way from start to end destination. Then the shortest path is only valid if it satisfies the constraint.</p> <h2>The MIT Map</h2> <p><a href="/c4x/MITx/6.00.2_2x/asset/files_ps10_files_ps10_campusmap.gif" target="_blank"><img class="aligncenter size-full wp-image-1162" title="ps10_campusmap" alt="" src="/c4x/MITx/6.00.2_2x/asset/files_ps10_files_ps10_campusmap.gif" height="275" width="468" /></a></p> <p>Here is the map of the MIT campus. From the text input file, <code>mit_map.txt</code>, you will build a representation of this map in Python using the graph-related data structures that we provide.</p> <p>Each line in <code>mit_map.txt</code> has 4 pieces of data in it in the following order separated by a single space (space-delimited): the start building, the destination building, the distance in meters between the two buildings, and the distance in meters between the two buildings that must be spent outdoors. For example, suppose the map text file contained the following line:</p> <pre>10 32 200 40 </pre> <p>This means that the map contains an edge from building 10 (start location) to building 32 (end location) that is 200 meters long, where 40 of those 200 meters are spent outside.</p> <p>To make the problem interesting, we will say that not every route between a pair of buildings is bi-directional. For example, it may be possible to get from building 54 (Green building) to building 56, but not the other way around, because the wind that blows away from the Green building is too strong.</p>
